# 모바일 앱 설계

Figma 원본:

- [Money Note Mobile App](https://www.figma.com/design/ys5W1k5bwO7YQi7sWWTZK3)

이 문서는 모바일 앱 구현 시 기준으로 삼을 화면 구조와 제품 원칙을 정리한다.

## 제품 방향

Money Note 모바일 앱은 웹 앱의 축소판이 아니다.

모바일 앱의 목적은 돈을 쓴 직후 5~10초 안에 기록하고, 결제일 전후에 카드대금을 빠르게 처리하며, 가족 관련 정산 상태를 확인하는 것이다.

우선순위:

1. 빠른 입력
2. 카드대금 처리
3. 청구와 가족카드 정산
4. 현재 상태 확인

복잡한 분석, 관리 로그, 복원 같은 기능은 일상 흐름을 방해하지 않도록 상태 화면의 고급 기능 영역이나 설정 화면에 둔다.

## 문구 원칙

모든 사용자 노출 문구는 한국어로 작성한다.

예시:

- `청구`
- `가족카드`
- `상태`
- `익월 유동성`
- `스냅샷 다운로드`

영어 기능명을 그대로 노출하지 않는다.

## 디자인 원칙

- 카드형 화면 구조를 사용한다.
- 표를 사용하지 않는다.
- 리스트와 카드 중심으로 표시한다.
- 한 손 조작을 우선한다.
- 금액은 크게 표시한다.
- 정보 밀도는 높게 유지하되 입력 흐름은 짧게 둔다.
- 애니메이션은 최소화한다.
- 장식보다 실용성을 우선한다.

## 하단 탭

하단 탭은 네 개만 둔다.

- 입력
- 결제
- 가족
- 상태

## 입력 화면

앱 실행 시 기본 화면이다.

상단에는 현재 월, 카드대금, 익월 유동성을 표시한다.

주요 카드:

- 오늘 지출 입력

입력 항목:

- 사용처
- 사용항목
- 금액

버튼:

- 지출 추가
- 알림에서 가져오기

최근 입력 목록은 최근 5건만 카드 리스트로 표시한다.

목표는 사용자가 10초 안에 지출을 기록하는 것이다.

## 결제 화면

카드대금 처리 전용 화면이다.

상단에는 남은 결제액과 결제일까지 남은 날짜를 표시한다.

자동 배분 카드:

- 처리 가능액 입력
- 날짜순 자동 배분

결제 항목은 카드 형태로 표시한다.

항목 카드에는 다음 정보를 둔다.

- 사용처 또는 대표 항목명
- 원금
- 남은 금액
- 즉시결제 버튼
- 이월 버튼

표 형태는 사용하지 않는다.

## 가족 화면

상단 전환으로 `청구`와 `가족카드`를 오간다.

청구 화면:

- 원금 합계
- 실청구 합계
- 공유하기
- 일괄 처리 완료
- 항목별 원금, 할인, 실청구, 삭제

가족카드 화면:

- 원금 합계
- 실결제 합계
- 가족카드 혜택 상태
- 공유하기
- 일괄 처리 완료

청구와 가족카드는 원장이 아니라 회수 예정 정보다. 따라서 모바일에서도 소비 통계나 유동성의 핵심 흐름처럼 보이지 않도록 분리한다.

## 상태 화면

조회 중심 화면이다.

표시 항목:

- 카드대금
- 월 지출
- 익월 유동성
- 동결
- 예산심사위원회 문구

하단 주요 버튼:

- 동기화
- 스냅샷 다운로드

고급 기능 영역:

- 월마감
- 백업
- 복원
- 관리 로그
- 설정

고급 기능은 접근 가능해야 하지만, 일상 입력 흐름을 방해하지 않아야 한다.

## 향후 기능 공간

모바일 앱은 다음 기능을 염두에 둔다.

- 카드 문자 알림 수집
- 카드 알림 자동 인식
- 자동 지출 생성
- 빠른 확인 처리

이 기능들은 입력 화면의 `알림에서 가져오기` 흐름과 자연스럽게 연결한다.
