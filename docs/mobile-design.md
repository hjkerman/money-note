# 모바일 앱 설계

Figma 원본:

- [Money Note Mobile App](https://www.figma.com/design/ys5W1k5bwO7YQi7sWWTZK3)

이 문서는 모바일 앱 구현 시 기준으로 삼을 화면 구조와 제품 원칙을 정리한다.

## 제품 방향

Money Note 모바일 앱은 웹 앱의 축소판이 아니다.

모바일 앱의 목적은 돈을 쓴 직후 5~10초 안에 기록하고, 현금흐름을 빠르게 보정하며, 가족 관련 정산 상태를 확인하는 것이다.

우선순위:

1. 빠른 입력
2. 현금흐름 관리
3. 한 달 지출 내역 확인
4. 청구와 가족카드 정산
5. 현재 상태 확인

복잡한 분석, 관리 로그, 복원 같은 기능은 일상 흐름을 방해하지 않도록 상태 화면의 고급 기능 영역이나 설정 화면에 둔다.
모바일 앱은 서버 DB를 원본으로 사용하므로 앱 시작 시 서버에 닿지 않으면 종료 안내를 보여준다.

## 문구 원칙

모든 사용자 노출 문구는 한국어로 작성한다.

예시:

- `청구`
- `가족카드`
- `상태`
- `익월 유동성`
- `스냅샷 다운로드`

영어 기능명을 그대로 노출하지 않는다.

## 디자인 원칙

- 카드형 화면 구조를 사용한다.
- 표를 사용하지 않는다.
- 리스트와 카드 중심으로 표시한다.
- 한 손 조작을 우선한다.
- 금액은 크게 표시한다.
- 정보 밀도는 높게 유지하되 입력 흐름은 짧게 둔다.
- 애니메이션은 최소화한다.
- 장식보다 실용성을 우선한다.

## 주 검증 기기

우선 검증 타깃은 Galaxy S23 FE / One UI 8.5다.

이 기기 기준으로 다음을 우선 확인한다.

- 한 손 입력이 불편하지 않은지
- 하단 탭과 입력 버튼이 시스템 내비게이션과 겹치지 않는지
- 알림 접근 권한 화면으로 정상 이동하는지
- 우리카드 승인 알림이 본인카드/가족카드 후보함으로 잘 분류되는지
- 최근 우리카드 알림 로그에서 파싱 실패와 할부 승인 원문을 확인할 수 있는지
- 큰 글꼴 설정에서 금액과 버튼 문구가 깨지지 않는지

## 하단 탭

하단 탭은 다섯 개를 둔다.

- 입력
- 현금
- 내역
- 정산
- 상태

## 입력 화면

앱 실행 시 기본 화면이다.

상단에는 현재 월, 카드대금, 익월 유동성을 표시한다.

상단 금액 아래에는 `오늘의 예산심사위원회` 카드를 둔다.
이 카드는 서버의 기존 Judgment 응답 중 대표 문구를 짧게 표시하며, 별도 판단 로직을 만들지 않는다.
카드를 누르면 전체 판단을 볼 수 있는 상태 탭으로 이동한다.

주요 카드:

- 카드 지출 입력

입력 항목:

- 사용처
- 사용항목
- 금액
- 할인 적용 여부

버튼:

- 지출 추가
- 알림에서 가져오기

최근 입력 목록은 최근 5건만 카드 리스트로 표시한다.

목표는 사용자가 10초 안에 지출을 기록하는 것이다.

## 내역 화면

현재 월의 지출 내역 전체를 카드 리스트로 보여준다.

원칙:

- 전체 기록이 아니라 당월 지출만 조회한다.
- 표를 쓰지 않는다.
- 사용처, 사용항목, 금액, 사용일을 카드 안에 표시한다.
- 교통, 하이패스, 통행료처럼 결제 정책상 눈에 띄어야 하는 항목은 작은 배지로 표시할 수 있다.

이 화면은 편집보다 확인을 우선한다.

## 현금 화면

현금흐름 관리 전용 화면이다.

상단에는 현재 유동성과 익월 유동성을 표시한다.

현금 입력 카드:

- 입금/출금 전환
- 내용
- 금액
- 이달 기준 수입 여부

현금흐름 항목은 카드 형태로 표시한다.

항목 카드에는 다음 정보를 둔다.

- 발생일
- 내용
- 입금/출금 금액
- 이달 기준 수입 표시
- 삭제 버튼

표 형태는 사용하지 않는다.

## 정산 화면

상단 카드 전환으로 `청구`와 `가족카드`를 오간다.

청구 화면:

- 원금 합계
- 실청구 합계
- 공유하기
- 일괄 처리 완료
- 항목별 원금, 할인, 실청구, 삭제
- 등록 전 할인 적용 여부 선택

가족카드 화면:

- 원금 합계
- 실결제 합계
- 가족카드 혜택 상태
- 공유하기
- 일괄 처리 완료
- 등록 전 할인 적용 여부 선택

청구와 가족카드는 원장이 아니라 회수 예정 정보다. 따라서 모바일에서도 소비 통계나 유동성의 핵심 흐름처럼 보이지 않도록 분리한다.

## 상태 화면

조회 중심 화면이다.

표시 항목:

- 카드대금
- 월 지출
- 익월 유동성
- 동결
- 예산심사위원회 문구
- 카드 한도 감시 문구
- 파산심사위원회 문구

하단 관리 진입:

- 관리 열기

관리 화면:

- 동결 금액
- 현금성 고정지출
- 카드 정기결제
- 월마감
- 백업 / 복원
- 설정

관리 기능은 접근 가능해야 하지만, 일상 입력 흐름을 방해하지 않아야 한다.

## 향후 기능 공간

모바일 앱은 다음 기능을 염두에 둔다.

- 카드 문자 알림 수집
- 카드 알림 자동 인식
- 자동 지출 생성
- 빠른 확인 처리

이 기능들은 입력 화면의 `알림에서 가져오기` 흐름과 자연스럽게 연결한다.

현재 Android 구현 원칙:

- `com.wooricard.smartapp` 알림만 처리한다.
- 일시불 승인 알림은 로컬 후보함에 저장한다.
- 알림 수신 즉시 서버 등록은 하지 않는다.
- 사용자가 후보를 확인하고 `등록`을 눌렀을 때만 기존 API를 호출한다.
- 할부 승인, 파싱 실패, 광고/혜택 안내는 후보로 만들지 않고 최근 우리카드 알림 로그에 남긴다.
- 최근 우리카드 알림 로그는 30건만 유지한다.
- 포맷 변경 대응을 위해 최근 로그 화면에서 rawText와 parsed result를 공유할 수 있게 둔다.
