# 수동 테스트 체크리스트

이 문서는 사용자가 웹 화면을 직접 누르면서 `money-note`의 주요 기능을 확인하기 위한 체크리스트다.

## 테스트 전 주의

- [ ] 중요한 실제 데이터가 있다면 DB와 Excel export를 백업했다.
- [ ] 삭제, 초기화, 월마감 테스트에는 구분하기 쉬운 테스트 항목만 사용한다.
- [ ] API 서버가 실행 중이며 `http://127.0.0.1:18080/health`가 정상이다.
- [ ] 웹 앱 `http://127.0.0.1:5173`을 열었다.

## 로그인

- [ ] 로그인 전에는 가계부 조작 화면이 보이지 않는다.
- [ ] 잘못된 비밀번호를 입력하면 raw JSON 대신 `아이디 또는 비밀번호가 맞지 않습니다.`가 보인다.
- [ ] 올바른 계정으로 로그인하면 요약과 큰 탭들이 보인다.
- [ ] 새로고침 후에도 로그인 상태가 유지된다.
- [ ] 로그아웃하면 로그인 화면으로 돌아간다.

## 당월 지출

- [ ] `당월 > 당월 지출`에서 날짜, 사용처, 사용항목, 금액을 입력해 추가할 수 있다.
- [ ] 추가한 적요가 `[사용처] 사용항목` 형태로 보인다.
- [ ] 추가 후 요약의 카드대금이 갱신된다.
- [ ] 지출 행을 삭제할 수 있다.
- [ ] 분류를 변경하면 상단에 `변경 사항 저장` 버튼이 나타난다.
- [ ] 분류 저장 후 새로고침해도 값이 유지된다.
- [ ] 저장하지 않은 분류 변경이 있을 때 페이지를 나가면 경고가 나타난다.

## 청구와 타인정산

- [ ] 청구와 타인정산 각각 항목을 추가할 수 있다.
- [ ] 테이블과 입력창이 하나의 panel 안에 붙어 있다.
- [ ] 각 행을 개별 삭제할 수 있다.
- [ ] `초기화`는 현재 탭의 항목만 삭제한다.
- [ ] 청구에는 자동 분류 결과가 표시된다.
- [ ] 타인정산에는 가족카드 한도 감시 문구가 표시된다.

## 고정지출과 카드 정기결제

- [ ] 큰 탭 이름은 `고정지출`이다.
- [ ] 내부에 `현금성 고정지출`과 `카드 정기결제`가 함께 보인다.
- [ ] 현금성 고정지출을 추가하고 삭제할 수 있다.
- [ ] 카드 정기결제에 결제일, 사용처, 사용항목, 금액을 입력할 수 있다.
- [ ] 카드 정기결제의 `확인`을 누르면 같은 구조의 당월 지출이 만들어진다.
- [ ] 카드 정기결제 항목을 삭제할 수 있다.

## 동결과 할부

- [ ] 동결 항목을 추가하고 삭제할 수 있다.
- [ ] 동결의 `확인`을 누르면 안내창이 나타난다.
- [ ] 승인하면 동결에서 사라지고 당월 지출에 추가된다.
- [ ] 할부액, 수수료율, 개월수를 입력해 할부를 추가할 수 있다.
- [ ] 수수료와 월 납입액이 원 단위 올림으로 계산된다.
- [ ] 활성 할부의 월 납입액이 카드대금 요약에 반영된다.

## 현금흐름과 주 수입

- [ ] 입금과 출금을 각각 추가할 수 있다.
- [ ] 출금은 음수, 입금은 양수로 표시된다.
- [ ] 현금흐름 추가 후 유동성 현황이 갱신된다.
- [ ] 입금 항목만 `주 수입`으로 표시할 수 있다.
- [ ] 주 수입 항목에는 `주 수입` 배지가 나타난다.
- [ ] 현금흐름 행을 삭제할 수 있다.

## 이번달 결제

- [ ] 대상 기간이 직전월 1일~말일로 표시된다.
- [ ] 직전월 사용내역이 날짜 오름차순으로 표시된다.
- [ ] 적요에 `교통` 문자열이 있는 항목에는 교통 배지가 나타난다.
- [ ] 각 항목의 원래 금액, 즉시결제 누적, 할인 누적, 남은 금액이 보인다.
- [ ] 원래 사용금액은 결제 처리 후에도 변하지 않는다.
- [ ] 원하는 항목을 직접 선택하거나 선택 해제할 수 있다.
- [ ] 선택 항목의 이번 처리액을 남은 금액보다 작은 일부 금액으로 직접 입력할 수 있다.
- [ ] 자동 배분 한도를 입력하고 `날짜순 자동 배분`을 누르면 앞 항목부터 배분된다.
- [ ] 마지막 선택 항목은 한도에 맞춰 일부 금액만 배분된다.
- [ ] 자동 배분 한도를 비우면 현재 유동성이 기본 한도로 사용된다.
- [ ] 즉시결제 반영 후 같은 금액의 현금흐름 출금이 생성된다.
- [ ] 즉시결제 기록을 취소하면 연결된 현금흐름도 함께 삭제된다.
- [ ] `할인액 처리`를 체크하고 수기 할인액을 반영할 수 있다.
- [ ] 할인액은 남은 결제금액만 줄이고 현금흐름을 만들지 않는다.
- [ ] 이번 달 주 수입이 있으면 심사 기준 수입으로 사용된다.
- [ ] 주 수입이 없으면 기본 심사 기준액이 사용된다.
- [ ] 결제 화면에서 기본 심사 기준액을 변경할 수 있다.
- [ ] 남은 결제액과 14일까지 남은 날짜에 따라 파산심사위원회 문구가 달라진다.

## 결제일 경과 처리

날짜 변경이 필요한 항목이므로 별도 테스트 DB 또는 개발 환경에서 확인한다.

- [ ] 익월 14일에는 즉시결제와 할인액 처리가 가능하다.
- [ ] 익월 15일부터는 즉시결제와 할인액 처리가 차단된다.
- [ ] 15일 이후 기록상 미결제액이 있어도 화면의 실질 미결제액은 0원으로 의제된다.
- [ ] `결제 안 된 내역 있습니다. 유동성 현황을 재설정하세요.` 안내가 나타난다.
- [ ] 실제 유동성을 수동 보정한 뒤 `유동성 보정 완료`를 누르면 안내가 사라진다.

## 가족 공유 PIN과 카카오톡

- [ ] 새 DB에서 청구/타인정산 공유 링크를 열면 기본 PIN `0000` 입력 화면이 나타난다.
- [ ] 기본 PIN `0000`을 입력하면 공유 페이지에 정상 접근할 수 있다.
- [ ] 기본 PIN 상태에서 본체에 로그인하면 PIN 변경 경고 문구와 `지금 PIN 변경` 버튼이 나타난다.
- [ ] 기본 PIN `0000`을 다시 설정해도 변경 경고가 유지된다.
- [ ] 본체 웹 상단의 `공유 PIN 설정`에서 숫자 네 자리를 설정할 수 있다.
- [ ] 기본값이 아닌 PIN 설정 후 본체의 PIN 변경 경고가 사라진다.
- [ ] PIN 설정 후 공유 링크를 새 브라우저에서 열면 PIN 입력 화면이 나타난다.
- [ ] 잘못된 PIN은 거부된다.
- [ ] 올바른 PIN 입력 후 공유 내용이 열린다.
- [ ] 카카오톡 인앱 브라우저에서도 PIN 입력과 공유 화면 열기가 정상이다.
- [ ] 카카오톡에서 링크를 다시 열었을 때 사이트 데이터가 유지되어 있다면 PIN을 다시 요구하지 않는다.
- [ ] 본체 웹에서 PIN을 변경하면 기존 공유 세션은 다시 PIN을 요구한다.
- [ ] 공유 화면에는 계좌번호, 송금 링크, 개인정보가 보이지 않는다.

## 통계와 월별 기록

- [ ] `통계 보기`를 누르면 소비 통계와 월별 기록이 함께 나타난다.
- [ ] 분류 저장 전 변경값도 통계에 즉시 반영된다.
- [ ] 월 선택을 바꾸면 해당 월 기록과 합계가 보인다.
- [ ] 통계와 판단 문구 외의 기본 조작 UI에는 불필요한 유머 문구가 없다.

## Excel export와 월마감

- [ ] Excel export를 실행하면 파일이 생성된다.
- [ ] export 파일의 원본 과거 hard data가 보존된다.
- [ ] 새 당월 지출은 `[사용처] 사용항목` 형식으로 export된다.
- [ ] 동적 테이블의 행 수와 요약 수식 범위가 맞는다.
- [ ] 테스트 데이터만 있는 상태에서 월마감을 실행한다.
- [ ] 당월 일반 지출이 전체 기록으로 이동한다.
- [ ] 카드 정기결제는 다음 달에도 남는다.
- [ ] 월마감 전후에도 카드 결제 배분 상태가 유지된다.

## 최종 확인

- [ ] 웹 개발자 콘솔에 처리되지 않은 오류가 없다.
- [ ] API 서버 로그에 반복 오류가 없다.
- [ ] 테스트 과정에서 만든 임시 항목을 모두 삭제했다.
- [ ] 중요한 실제 데이터가 의도하지 않게 바뀌지 않았다.
