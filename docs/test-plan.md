# 수동 테스트 체크리스트

이 문서는 사용자가 웹 화면을 직접 누르면서 `money-note`의 주요 기능을 확인하기 위한 체크리스트다.

## 테스트 전 주의

- [ ] 중요한 실제 데이터가 있다면 `CSV 백업`으로 CSV 덤프 파일을 받아 두었다.
- [ ] 삭제, 초기화, 월마감 테스트에는 구분하기 쉬운 테스트 항목만 사용한다.
- [ ] API 서버가 실행 중이며 `http://127.0.0.1:18080/health`가 정상이다.
- [ ] 웹 앱 `http://127.0.0.1:5173`을 열었다.

## 로그인

- [ ] 로그인 전에는 가계부 조작 화면이 보이지 않는다.
- [ ] 잘못된 비밀번호를 입력하면 raw JSON 대신 `아이디 또는 비밀번호가 맞지 않습니다.`가 보인다.
- [ ] 올바른 계정으로 로그인하면 요약과 큰 탭들이 보인다.
- [ ] 새로고침 후에도 로그인 상태가 유지된다.
- [ ] 로그아웃하면 로그인 화면으로 돌아간다.

## 당월 지출

- [ ] `당월 > 당월 지출`에서 날짜, 사용처, 사용항목, 금액을 입력해 추가할 수 있다.
- [ ] 항목을 추가하면 날짜가 아닌 첫 입력 칸으로 포커스가 돌아간다.
- [ ] 추가한 적요가 `[사용처] 사용항목` 형태로 보인다.
- [ ] 추가 후 요약의 카드대금이 갱신된다.
- [ ] 지출 행을 삭제할 수 있다.
- [ ] 분류를 변경하면 상단에 `변경 사항 저장` 버튼이 나타난다.
- [ ] 분류 저장 후 새로고침해도 값이 유지된다.
- [ ] 저장하지 않은 분류 변경이 있을 때 페이지를 나가면 경고가 나타난다.
- [ ] 본인회원 카드의 월별 할인 혜택을 `미정`, `혜택 있음`, `혜택 없음`으로 설정할 수 있다.
- [ ] 혜택 있음 또는 미정인 달에는 카드 지출에 기본 1.2% 할인이 보인다.
- [ ] `할인 제외`를 누르면 해당 항목 할인액은 0원으로 계산된다.
- [ ] 제외한 항목에서 `할인 적용`을 누르면 기본 1.2% 할인 계산으로 돌아간다.
- [ ] 본인회원 카드 혜택을 `혜택 없음`으로 설정하면 할인액은 전부 0원으로 계산된다.
- [ ] 할인 미확인 여부 때문에 붉은 배경이 표시되지 않는다.

## 임시 메모

- [ ] 화면 오른쪽 아래의 `메모` 버튼으로 임시 메모를 펼치고 접을 수 있다.
- [ ] 메모를 접었다 다시 펼치면 작성 중인 내용이 남아 있다.
- [ ] `비우기`를 누르면 작성 내용이 사라진다.
- [ ] 새로고침하면 임시 메모 내용이 사라진다.

## 청구와 타인정산

- [ ] 청구와 타인정산 각각 항목을 추가할 수 있다.
- [ ] 테이블과 입력창이 하나의 panel 안에 붙어 있다.
- [ ] 각 행을 개별 삭제할 수 있다.
- [ ] `초기화`는 현재 탭의 항목만 삭제한다.
- [ ] 청구에는 자동 분류 결과가 표시된다.
- [ ] 타인정산에는 가족카드 한도 감시 문구가 표시된다.
- [ ] 청구는 본인회원 카드 할인 혜택 설정을, 타인정산은 별도의 가족카드 할인 혜택 설정을 보여준다.
- [ ] 같은 달에도 본인회원 카드와 가족카드의 할인 혜택 여부를 서로 다르게 설정할 수 있다.
- [ ] 청구 항목은 원래 금액과 할인 후 실 결제액을 함께 확인할 수 있다.
- [ ] 청구 통계와 합계는 원래 금액보다 실 결제액을 우선한다.
- [ ] 청구 금액은 익월 유동성 계산에 반영되지 않는다.
- [ ] 청구 공유 페이지에도 할인액과 할인 후 실제 청구 합계가 보인다.
- [ ] 청구와 타인정산은 사용일을 입력할 수 있고 사용일순으로 정렬된다.

## 고정지출과 카드 정기결제

- [ ] 큰 탭 이름은 `고정지출`이다.
- [ ] 내부에 `현금성 고정지출`과 `카드 정기결제`가 함께 보인다.
- [ ] 현금성 고정지출을 추가하고 삭제할 수 있다.
- [ ] 카드 정기결제에 결제일, 사용처, 사용항목, 금액을 입력할 수 있다.
- [ ] 카드 정기결제 목록에서도 사용처와 사용항목이 서로 다른 열로 보인다.
- [ ] 카드 정기결제는 결제일순으로 표시되고, 사용항목이 비어 있으면 화면에 `좌동`으로 보인다.
- [ ] 카드 정기결제에서 사용항목만 비워둘 수 있고, 결제일·사용처·금액이 없으면 추가되지 않는다.
- [ ] 카드 정기결제도 카드 지출로 편입되면 같은 할인 정책을 따른다.
- [ ] 카드 정기결제의 `확인`을 누르면 같은 구조의 당월 지출이 만들어진다.
- [ ] 카드 정기결제 항목을 삭제할 수 있다.

## 동결과 할부

- [ ] 동결 항목을 추가하고 삭제할 수 있다.
- [ ] 동결 항목에는 당월 지출 자동 편입 버튼이 없고 개별 삭제만 가능하다.
- [ ] 승인하면 동결에서 사라지고 당월 지출에 추가된다.
- [ ] 할부액, 수수료율, 개월수를 입력해 할부를 추가할 수 있다.
- [ ] 수수료와 월 납입액이 원 단위 올림으로 계산된다.
- [ ] 활성 할부의 월 납입액이 카드대금 요약에 반영된다.

## 현금흐름과 주 수입

- [ ] 입금과 출금을 각각 추가할 수 있다.
- [ ] 출금은 음수, 입금은 양수로 표시된다.
- [ ] 현금흐름 추가 후 유동성 현황이 갱신된다.
- [ ] 입금 항목만 `주 수입`으로 표시할 수 있다.
- [ ] 주 수입 항목에는 `주 수입` 배지가 나타난다.
- [ ] 현금흐름 행을 삭제할 수 있다.

## 이번달 결제

- [ ] 대상 기간이 직전월 1일~말일로 표시된다.
- [ ] 직전월 사용내역이 날짜 오름차순으로 표시된다.
- [ ] 적요에 `교통` 문자열이 있는 항목에는 교통 배지가 나타난다.
- [ ] 모든 카드 사용내역에 `이월` 버튼이 나타난다.
- [ ] 하이패스/통행료 항목이 여러 건이면 결제 대상 목록에서 하나의 통합 행으로 보인다.
- [ ] 통행료/하이패스 항목도 일부 즉시결제할 수 있다.
- [ ] 통합 행에 일부 금액을 입력하면 원본 하이패스/통행료 항목에 순서대로 배분된다.
- [ ] 자동 배분 잔액이 통행료보다 적으면 남은 예산만큼 일부 배분한다.
- [ ] 카드 사용내역의 `이월`을 누르면 현재 결제 합계와 자동 배분 대상에서 제외된다.
- [ ] 이월한 항목은 당월 사용내역 맨 앞에 날짜 없이 `[이월]` 적요로 나타난다.
- [ ] 같은 결제월에 `이번 달에 처리`를 누르면 원래 장부 날짜, 적요, 위치가 복원된다.
- [ ] 14일까지는 이월과 `이번 달에 처리` 전환이 가능하다.
- [ ] 15일부터는 이월이 확정되어 `이번 달에 처리`로 되돌릴 수 없다.
- [ ] 다음 결제월에는 이월 항목이 날짜 없이 `[이월]` 적요로 결제 대상 맨 앞에 나타난다.
- [ ] 다음 결제월의 이월 항목에는 재이월 버튼이 나타나지 않는다.
- [ ] `전월 매입 지연 보정`에서 직전월 날짜, 사용처, 사용항목, 금액을 입력해 추가할 수 있다.
- [ ] 전월 보정으로 추가한 항목은 즉시 이번달 결제 대상에 나타난다.
- [ ] 직전월이 아닌 날짜는 전월 보정에 추가할 수 없다.
- [ ] 결제 대상 목록에서 사용내역을 삭제할 수 있다.
- [ ] 통합 행을 삭제하면 그 안의 원본 하이패스/통행료 항목이 함께 삭제된다.
- [ ] 카드사 환급은 과거 기록을 삭제하지 않고 현금흐름 입금으로 기록한다.
- [ ] 각 항목의 원래 금액, 즉시결제 누적, 할인 누적, 남은 금액이 보인다.
- [ ] 원래 사용금액은 결제 처리 후에도 변하지 않는다.
- [ ] 원하는 항목을 직접 선택하거나 선택 해제할 수 있다.
- [ ] 선택 항목의 이번 처리액을 남은 금액보다 작은 일부 금액으로 직접 입력할 수 있다.
- [ ] 자동 배분 한도를 입력하고 `날짜순 자동 배분`을 누르면 앞 항목부터 배분된다.
- [ ] 마지막 선택 항목은 한도에 맞춰 일부 금액만 배분된다.
- [ ] 자동 배분 한도를 비우면 현재 유동성이 기본 한도로 사용된다.
- [ ] 즉시결제 반영 후 같은 금액의 현금흐름 출금이 생성된다.
- [ ] 즉시결제 기록을 취소하면 연결된 현금흐름도 함께 삭제된다.
- [ ] 할인액은 남은 결제금액만 줄이고 현금흐름을 만들지 않는다.
- [ ] 이번 달 주 수입이 있으면 심사 기준 수입으로 사용된다.
- [ ] 주 수입이 없으면 기본 심사 기준액이 사용된다.
- [ ] 결제 화면에서 기본 심사 기준액을 변경할 수 있다.
- [ ] 남은 결제액과 14일까지 남은 날짜에 따라 파산심사위원회 문구가 달라진다.
- [ ] 같은 결제 상태에서 새로고침해도 파산심사위원회 문구가 임의로 바뀌지 않는다.
- [ ] 같은 위험 등급에서도 결제일까지 남은 날짜 또는 미결제액을 바꾸면 다른 문구가 나타날 수 있다.

## 결제일 경과 처리

날짜 변경이 필요한 항목이므로 별도 테스트 DB 또는 개발 환경에서 확인한다.

- [ ] 익월 14일에는 즉시결제가 가능하다.
- [ ] 익월 15일부터는 즉시결제가 차단된다.
- [ ] 15일 이후 기록상 미결제액이 있어도 화면의 실질 미결제액은 0원으로 의제된다.
- [ ] `결제 안 된 내역 있습니다. 유동성 현황을 재설정하세요.` 안내가 나타난다.
- [ ] 실제 유동성을 수동 보정한 뒤 `유동성 보정 완료`를 누르면 안내가 사라진다.

## 가족 공유 PIN과 카카오톡

- [ ] 새 DB에서 청구/타인정산 공유 링크를 열면 기본 PIN `0000` 입력 화면이 나타난다.
- [ ] 기본 PIN `0000`을 입력하면 공유 페이지에 정상 접근할 수 있다.
- [ ] 기본 PIN 상태에서 본체에 로그인하면 PIN 변경 경고 문구와 `지금 PIN 변경` 버튼이 나타난다.
- [ ] 기본 PIN `0000`을 다시 설정해도 변경 경고가 유지된다.
- [ ] 본체 웹 상단의 `공유 PIN 설정`에서 숫자 네 자리를 설정할 수 있다.
- [ ] 기본값이 아닌 PIN 설정 후 본체의 PIN 변경 경고가 사라진다.
- [ ] PIN 설정 후 공유 링크를 새 브라우저에서 열면 PIN 입력 화면이 나타난다.
- [ ] 잘못된 PIN은 거부된다.
- [ ] 올바른 PIN 입력 후 공유 내용이 열린다.
- [ ] 카카오톡 인앱 브라우저에서도 PIN 입력과 공유 화면 열기가 정상이다.
- [ ] 카카오톡에서 링크를 다시 열었을 때 사이트 데이터가 유지되어 있다면 PIN을 다시 요구하지 않는다.
- [ ] 본체 웹에서 PIN을 변경하면 기존 공유 세션은 다시 PIN을 요구한다.
- [ ] 공유 화면에는 계좌번호, 송금 링크, 개인정보가 보이지 않는다.

## 통계와 월별 기록

- [ ] 요약 상단에 예산심사위원회 한 줄 평이 표시된다.
- [ ] 당월 지출, 현금흐름, 청구, 타인정산, 동결에 테스트 항목을 하나씩 추가할 때마다 상단 한 줄 평이 달라진다.
- [ ] 장부를 바꾸지 않고 새로고침하면 상단 한 줄 평이 유지된다.
- [ ] `통계 보기`를 누르면 소비 통계와 월별 기록이 함께 나타난다.
- [ ] 분류 저장 전 변경값도 통계에 즉시 반영된다.
- [ ] 연-월 선택을 바꾸면 해당 월의 구조화된 current/archive 기록과 합계가 보인다.
- [ ] 당월 지출과 월별 기록 표에서 사용처, 사용항목, 분류가 서로 다른 열로 보인다.
- [ ] 당월 지출에서 사용항목만 비워둘 수 있고, 날짜·사용처·금액이 없으면 추가되지 않는다.
- [ ] 금액 입력칸에는 일반 텍스트를 입력할 수 없고 정수 숫자만 허용된다.
- [ ] 현금흐름을 제외한 금액 입력/API에 음수를 넣으면 거부된다.
- [ ] 항목 추가 성공 후 날짜 칸은 건너뛰고 해당 폼의 첫 입력칸으로 focus가 돌아간다.
- [ ] 통계와 판단 문구 외의 기본 조작 UI에는 불필요한 유머 문구가 없다.
- [ ] 청구 공유 문구가 의료, 치과, 교통, 소액 다건 같은 청구 구성에 따라 달라진다.
- [ ] 의료 청구 2건 이하는 정기 유지보수인 안 아픈 달로 평가된다.
- [ ] 의료 청구 3건은 정기 진료 외에 한 번 아팠던 달로 평가된다.
- [ ] 의료 청구 4건 이상은 집에서 걱정할 만큼 많이 아픈 달로 평가된다.
- [ ] 타인정산 공유 문구가 합산 한도 사용률과 특정 고액 항목 집중도에 따라 달라진다.

## CSV 백업과 월마감

- [ ] `CSV 백업`을 누르면 `.csv` 파일이 다운로드된다.
- [ ] CSV 첫 줄에 `__table` 컬럼이 있고, 행마다 대상 테이블명이 들어 있다.
- [ ] 백업 파일을 `CSV 복원`으로 다시 가져오면 화면 데이터가 복원된다.
- [ ] CSV 복원 후 사용자 계정과 로그인 상태는 유지된다.
- [ ] 설정 모달의 장부 데이터 전체 초기화는 현재 비밀번호 없이는 실행되지 않는다.
- [ ] 초기화 후 계정, 로그인 세션, 공유 PIN, 예정 수입, 이자지출 설정은 유지된다.
- [ ] 테스트 데이터만 있는 상태에서 월마감을 실행한다.
- [ ] 새 달인데 이전 월 장부가 남아 있으면 첫 화면에 월마감 검토 경고가 나타난다.
- [ ] 26일까지는 현재 달 조기 월마감 버튼이 비활성화된다.
- [ ] 27일부터 현재 달 조기 월마감 확인창과 버튼이 활성화된다.
- [ ] 조기 월마감 후 같은 달 날짜로 지출을 추가하면 전체 기록에 추가되었다는 안내가 나타난다.
- [ ] 이전 월 말일 기록과 새 달 1일 기록을 모두 추가한 뒤 월마감한다.
- [ ] 가장 오래된 이전 월 일반 지출만 전체 기록으로 이동한다.
- [ ] 먼저 추가한 새 달 기록은 당월 기록에 그대로 남는다.
- [ ] 현재 달만 남은 상태에서는 월마감 버튼이 비활성화된다.
- [ ] 카드 정기결제는 다음 달에도 남는다.
- [ ] 월마감 전후에도 카드 결제 배분 상태가 유지된다.
- [ ] 청구 `일괄 처리 완료`는 청구 항목만 현재 목록과 공유 페이지에서 삭제한다.
- [ ] 타인정산 `일괄 처리 완료`는 타인정산 항목만 현재 목록과 공유 페이지에서 삭제한다.
- [ ] 청구 본인용 표에는 자동 분류 열이 보이지 않지만 공유 판단 문구는 정상적으로 나타난다.

## 관리 로그

- [ ] 항목 추가·수정·삭제 후 `관리 로그`에 HTTP 방식, 경로, 결과 코드가 나타난다.
- [ ] 관리 로그에는 요청 본문, 비밀번호, 세션 토큰이 보이지 않는다.
- [ ] `로그 초기화` 확인창에서 취소하면 로그가 유지된다.
- [ ] `로그 초기화`를 승인하면 전체 로그가 사라지고 초기화 요청 자체는 새 로그로 남지 않는다.

## 최종 확인

- [ ] 웹 개발자 콘솔에 처리되지 않은 오류가 없다.
- [ ] API 서버 로그에 반복 오류가 없다.
- [ ] 테스트 과정에서 만든 임시 항목을 모두 삭제했다.
- [ ] 중요한 실제 데이터가 의도하지 않게 바뀌지 않았다.
